PATH Hijacking


PATH Hijacking sirve para alterar la variable $PATH o jugar con ella para poder ejecutar binarios mediante rutas relativas

┌──(kali@kali)-[~]
└─$ echo $PATH
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/local/games:/usr/games


Ahora por ejemplo creamos un archivo llamado “whoami” en nuestro home y metemos dentro por ejemplo el comando "ps".

Con “export” podremos editar la variable $PATH, en este caso le indicaremos que queremos que la ruta prioritaria sea nuestro home que es el directorio en el que estamos ahora y despues las demas rutas que ya estaban en el $PATH

┌──(kali@kali)-[~]
└─$ export PATH=/home/prueba:$PATH


Ahora el sistema ejecutara whoami y nos mostrara un ps ya que ejecutara el whoami de nuestro directorio home antes que el que esta en /usr/bin
